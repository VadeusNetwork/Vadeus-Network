# Contributing to Vadeus Network

We love your contributions! To maintain the high performance and security of the Vadeus protocol, please follow these guidelines:

## Git Commit Standards
We use **Conventional Commits**. Your commit messages should look like this:
- `feat: add solana staking support`
- `fix: resolve reward calculation overflow`
- `docs: update tokenomics distribution`

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Ensure all tests pass in your local Docker environment.
3. Update the `docs/` if you're changing the protocol logic.
4. Your PR will be reviewed by the Core Team within 48 hours.

## Coding Standards
- Use TypeScript for all new modules.
- Ensure 100% test coverage for financial transactions.
- Follow the architectural patterns defined in `docs/ARCHITECTURE.md`.
